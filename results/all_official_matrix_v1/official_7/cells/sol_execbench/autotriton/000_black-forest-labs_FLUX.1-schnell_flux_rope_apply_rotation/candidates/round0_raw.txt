The RoPE rotation involves splitting the head dimension into pairs and applying a complex rotation formula to each pair. The goal is to leverage Triton for efficient GPU computation while ensuring numerical accuracy and performance.

### Approach
1. **Problem Analysis**: The reference model applies a complex rotation to each pair of features in the head dimension. The rotation formula is:
   - \( \text{out1} = x1 \times \text{cos1} - x2 \times \text{sin1} \)
   - \( \text{out2} = x1 \times \text{sin2} + x2 \times \text{cos2} \)
   where \( x1 \) and \( x2 \) are consecutive features in the head dimension, and \( \text{cos1}, \text{sin1}, \text{cos2}, \text{sin2} \) are corresponding frequency values.

2. **Triton Kernel Design**: 
   - **Grid Construction**: Launch a grid where each block processes one pair of features (two elements) across all batches, sequences, and heads.
   - **Memory Access**: Efficiently load input features and frequency values using computed strides to handle tensor dimensions correctly.
   - **Computation**: Perform the rotation using loaded values and store results back to the output tensor.

3. **Host Function**: 
   - Compute necessary strides for input tensors.
   - Determine grid size based on tensor dimensions.
   - Invoke the Triton kernel with appropriate parameters.

import torch
import triton
import triton.language as tl

@torch.no_grad()
def run(
    query_or_key: torch.Tensor,
    freqs_cos: torch.Tensor,
    freqs_sin: torch.Tensor,
) -> torch.Tensor:
    """
    Apply rotary position embeddings to query or key tensor using a Triton kernel.
    
    The rotation is applied using the formula:
    - Split features into pairs (x1, x2)
    - Rotate: (x1 * cos1 - x2 * sin1, x1 * sin2 + x2 * cos2)
    
    Args:
        query_or_key: Input tensor of shape (batch, seq_len, num_heads, head_dim)
        freqs_cos: Cosine frequencies of shape (seq_len, head_dim)
        freqs_sin: Sine frequencies of shape (seq_len, head_dim)
        
    Returns:
        Rotated tensor of same shape as input
    """
    assert query_or_key.is_cuda, "Input tensor must be on CUDA device"
    assert freqs_cos.is_cuda and freqs_sin.is_cuda, "Frequency tensors must be on CUDA device"
    assert query_or_key.shape[-1] % 2 == 0, "Head dimension must be even"

    B, S, H, D = query_or_key.shape
    total_pairs = D // 2
    grid = (B * S * H * total_pairs,)

    # Compute strides
    stride_qb, stride_qs, stride_qh, stride_qd = query_or_key.stride()
    stride_fs, stride_fd = freqs_cos.stride()
    stride_ks, stride_kd = freqs_sin.stride()
    stride_ob, stride_os, stride_oh, stride_od = query_or_key.stride()

    # Launch kernel
    _rope_apply_rotation_kernel[grid](
        query_or_key, freqs_cos, freqs_sin, query_or_key,
        B, S, H, D,
        stride_qb, stride_qs, stride_qh, stride_qd,
        stride_fs, stride_fd,
        stride_ks, stride_kd,
        stride_ob, stride_os, stride_oh, stride_od,
        D,
        total_pairs,
    )
    return query_or_key
