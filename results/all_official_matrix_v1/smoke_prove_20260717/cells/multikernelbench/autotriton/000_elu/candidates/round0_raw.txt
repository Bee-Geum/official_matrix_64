To solve this problem, we need to implement an optimized GPU kernel using Triton for the ELU activation function, which matches the functionality of the given PyTorch reference model. The ELU function is defined as \( \text{ELU}(x) = x \) if \( x > 0 \) and \( \alpha \times (\exp(x) - 1) \) if \( x \leq 0 \). The solution involves creating a Triton kernel that processes elements in parallel, applying the ELU function efficiently, and integrating this kernel into a PyTorch module.

### Approach
1. **Triton Kernel Design**: The kernel processes elements in blocks for efficiency. Each block handles a contiguous set of elements, and within each block, elements are processed in parallel. The kernel uses a mask to avoid out-of-bounds memory accesses.
2. **Element-wise Processing**: For each element, the kernel checks if it is greater than zero. If true, the element remains unchanged; otherwise, it computes \( \alpha \times (\exp(x) - 1) \).
3. **PyTorch Module Integration**: The `ModelNew` class initializes with an alpha parameter and uses the Triton kernel in its forward pass. The input tensor is flattened to a 1D array for processing, and the output is reshaped to match the input dimensions. The kernel is launched with a grid size calculated based on the number of elements and a block size of 1024.

### Solution Code
```python
import torch
import torch.nn as nn
import triton
import triton.language as tl

@triton.jit
def elu_kernel(
    x_ptr,
    output_ptr,
    alpha,
    n_elements,
    BLOCK_SIZE: tl.constexpr,
):
    pid = tl.program_id(axis=0)
    block_start = pid * BLOCK_SIZE
    offsets = block_start + tl.arange(0, BLOCK_SIZE)
    mask = offsets < n_elements
    x = tl.load(x_ptr + offsets, mask=mask)
    condition = x > 0.0
    exp_term = tl.math.exp(x) - 1.0
    result = tl.where(condition, x, alpha * exp_term)
    tl.store(output_ptr + offsets, result, mask=mask)

class ModelNew(nn.Module):
    def __init__(self, alpha: float = 1.0):
        super(ModelNew, self).__init__()
        self.alpha = alpha

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        original_shape = x.shape
        x_flat = x.view(-1)
        n_elements = x_flat.numel()
        output_flat = torch.empty_like(x_flat)
        grid = lambda meta: (triton.cdiv(n_elements, meta['BLOCK_SIZE']),)
        elu_kernel[grid](x_flat, output_flat, self.alpha, n_elements, BLOCK_SIZE=1024)
        return output_flat.view(original_shape)
```

The kernel processes elements in blocks of 1024, ensuring efficient parallel execution.
2. **PyTorch Module**: The `ModelNew` class initializes with an alpha parameter, matching the interface of the reference model. The kernel launch configuration is dynamically calculated based on the number of elements to ensure all elements are processed.
